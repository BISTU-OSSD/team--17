# Contributing to DevSkillMapper

Thank you for your interest in contributing! We welcome contributions from everyone.

## How to Contribute

### Reporting Issues

- Search the [issue tracker](https://github.com/BISTU-OSSD/team--17/issues) first to avoid duplicates
- Provide a clear title and detailed description
- Include steps to reproduce, expected behavior, and actual behavior
- Attach screenshots if applicable

### Submitting Pull Requests

1. **Fork** the repository and create your branch from `main`
2. **Clone** your fork locally
3. Make your changes, ensuring code style consistency
4. **Test** your changes thoroughly
5. **Commit** with a clear and descriptive message
6. **Push** to your fork and open a Pull Request

### Pull Request Guidelines

- Keep PRs focused on a single concern
- Update documentation if your changes affect the API or user-facing behavior
- Ensure the PR passes all CI checks
- Reference any related issues in the PR description

## Code Style

- **Python**: Follow PEP 8. Use type hints. Run `ruff` or `black` for formatting
- **JavaScript/React**: Follow ESLint rules. Use functional components with hooks
- **CSS**: Use the existing BEM-like class naming and dark theme conventions

## Development Setup

See the [README.md](/README.md) for local development instructions.

## Code of Conduct

Be respectful and constructive. Harassment or disrespectful behavior will not be tolerated.

Thank you for helping make DevSkillMapper better!
